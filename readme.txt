========================================================================
NURSECONNECT LITE - แพลตฟอร์มจับคู่พยาบาลและผู้ดูแล
========================================================================

ยินดีต้อนรับสู่ NurseConnect Lite แพลตฟอร์มที่พัฒนาด้วย Flask แบบเรียบง่าย
เพื่อใช้ในการเชื่อมต่อผู้ป่วยกับพยาบาลวิชาชีพและผู้ดูแลมืออาชีพในประเทศไทย

--- 1. วิธีการเริ่มโปรแกรม ---
1. ตรวจสอบว่าคุณได้ติดตั้ง Python แล้ว (แนะนำเวอร์ชัน 3.10 ขึ้นไป)
2. ติดตั้งไลบรารีที่จำเป็น:
   pip install flask
3. รันโปรแกรม:
   python app.py
4. เปิดเบราว์เซอร์แล้วไปที่:
   http://127.0.0.1:3000

--- 2. โครงสร้างไฟล์และหน้าที่ ---

/NurseConnectLite
│
├── app.py                # ตรรกะหลักของระบบ จัดการเส้นทาง (Routes), เซสชัน, 
│                         # การยืนยันตัวตน และการจัดการไฟล์ CSV ทั้งหมด
│
├── templates/            # โฟลเดอร์เก็บเทมเพลต HTML (Jinja2)
│   ├── base.html         # โครงสร้างหลัก, การตั้งค่า CSS และแถบนำทาง
│   ├── index.html        # หน้าแรกของเว็บไซต์ แสดงหมวดหมู่บริการ
│   ├── login.html        # หน้าเข้าสู่ระบบสำหรับผู้ใช้ทุกบทบาท
│   ├── register.html     # หน้าลงทะเบียน แบ่งตามประเภทผู้ใช้
│   ├── nurses.html       # หน้าแสดงรายชื่อพยาบาลและผู้ดูแลที่เปิดให้จอง
│   ├── client_dashboard.html # ส่วนของผู้รับบริการ (จองงานและติดตามสถานะ)
│   ├── nurse_dashboard.html  # ส่วนของพยาบาล (รับงานและส่งมอบงาน)
│   └── admin_dashboard.html  # ส่วนของผู้ดูแลระบบ (ยืนยันตัวตนพยาบาล)
│
├── users.csv             # ฐานข้อมูลบัญชีผู้ใช้ (ชื่อ, อีเมล, บทบาท)
├── nurses.csv            # ฐานข้อมูลโปรไฟล์พยาบาลและเลขใบอนุญาต
└── bookings.csv          # ฐานข้อมูลการจองบริการและสถานะงาน

--- 3. บัญชีทดสอบ (ตัวอย่างข้อมูล) ---

คุณสามารถใช้บัญชีเหล่านี้เพื่อทดสอบฟังก์ชันการใช้งานต่างๆ:

ก) ผู้ดูแลระบบ (ADMIN)
   - อีเมล: admin@platform.com
   - รหัสผ่าน: admin1234
   - หน้าที่: ยืนยันตัวตนพยาบาล, ลบข้อมูล, ดูสรุปรายได้

ข) พยาบาล/ผู้ดูแล (NURSE)
   - อีเมล: nurse1@test.com (ยืนยันแล้ว)
   - อีเมล: nurse2@test.com (ยังไม่ได้ยืนยัน)
   - รหัสผ่าน: 1234
   - หน้าที่: รับงานที่ลูกค้าจองเข้ามา และอัปเดตสถานะงาน

ค) ผู้รับบริการ (CLIENT)
   - อีเมล: client1@test.com
   - รหัสผ่าน: 1234
   - หน้าที่: ค้นหาพยาบาล, ทำการจองบริการ และติดตามผล

--- 4. หมายเหตุทางเทคนิคสำหรับนักพัฒนา ---

- การเก็บข้อมูล: แอปนี้ใช้ไฟล์ CSV ในการเก็บข้อมูล โดยจะโหลดข้อมูลเข้าหน่วยความจำ
  ทุกครั้งที่มีคำขอ และจะเขียนไฟล์ใหม่ทั้งหมดทุกครั้งที่มีการแก้ไขข้อมูล
- การยืนยันตัวตน: ใช้ Flask Sessions และมีตัวตกแต่ง (@require_role) เพื่อจำกัดสิทธิ์
- ข้อมูลเริ่มต้น: หากไฟล์ users.csv ว่างเปล่า ระบบจะสร้างบัญชีทดสอบด้านบนให้โดยอัตโนมัติ
